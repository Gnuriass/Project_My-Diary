# The Locked Diary (C++ OOP Practice Project)

The Locked Diary is a console-based personal diary application developed in C++ to practice software design principles using **Object-Oriented Programming (OOP)** concepts. The project focuses on implementing secure diary management with password protection, data organization, and object-oriented design patterns.

---

## 🌟 Features

### Password Protection System

* Secure diary access using password authentication.
* Users must enter the correct password before accessing diary management functions.

### Diary Management

* Create diary entries with:

  * Title
  * Date
  * Content
* View all diary entries.
* Delete selected diary entries.

### Specialized Logging Features

#### Record Emotions

* Quickly record current emotional states through a dedicated menu.
* Supports 10 predefined emotions with emoji indicators, such as:

  * Happy 
  * Excited 
  * Tired 
  * Confident 
  * And more.

#### Record Meetings

* Separate menu for recording meeting notes and important appointments.

### Dynamic Settings

* Change the system password directly within the application.
* Improve security without modifying the source code.

---

## 🛠️ Object-Oriented Programming Concepts

This project was designed to demonstrate key OOP principles in C++.

### Encapsulation

* Data members are protected using `private` and `protected` access modifiers.
* Access is controlled through getter and setter methods.

### Inheritance

* `Diary` inherits from `DiaryEntry`.
* `LockedDiary` inherits from `Diary` and extends functionality with password protection mechanisms.

### Polymorphism & Method Overriding

* Uses virtual functions (`virtual`) to enable runtime polymorphism.
* Derived classes override inherited methods to customize behavior for password-secured operations.

## 🚀 Getting Started

* Code::Blocks IDE (recommended)

Open a terminal inside the project directory and run:

#### Windows


#### Linux / macOS


# The Locked Diary (C++ OOP Practice Project)

โปรเจคแอปพลิเคชันไดอารี่ส่วนตัวแบบล็อกรหัสผ่าน (Console-based Application) พัฒนาขึ้นโดยใช้ภาษา C++ เพื่อฝึกฝนทักษะการออกแบบสถาปัตยกรรมซอฟต์แวร์ด้วยแนวคิด **Object-Oriented Programming (OOP)** ---

## 🌟 คุณสมบัติของระบบ (Features)
- **Password Protection System:** ระบบล็อกไดอารี่ด้วยรหัสผ่านความปลอดภัยสูง ตรวจสอบสิทธิ์ (Authentication) ทุกครั้งก่อนเข้าถึงเมนูจัดการข้อมูล
- **Diary Management:** รองรับการบันทึกข้อมูลไดอารี่ทั่วไป (เพิ่มหัวข้อ, วันที่ และเนื้อหา), แสดงรายการไดอารี่ทั้งหมด และเลือกนำไดอารี่ที่ต้องการออกได้ (Delete Entry)
- **Specialized Loggers:** ฟังก์ชันบันทึกข้อมูลหมวดหมู่เฉพาะตัว:
  - **Record Emotions:** เลือกบันทึกสถานะอารมณ์ปัจจุบันจาก Quick Menu (มีตัวเลือกอารมณ์พร้อม Emoji ถึง 10 รูปแบบ เช่น Happy , Tired , Confident )
  - **Record Meeting:** เมนูแยกสำหรับจดบันทึกรายละเอียดการประชุมหรือการนัดหมายสำคัญ
- **Dynamic Settings:** มีฟังก์ชันสำหรับเปลี่ยนรหัสผ่านเพื่อความปลอดภัย (Change Password) ได้โดยตรงจากภายในโปรแกรม

---

## 🛠️ แนวคิด OOP ที่ใช้ในโปรเจค (Object-Oriented Concepts)
โครงสร้างโปรแกรมนี้ถูกออกแบบขึ้นโดยใช้คุณสมบัติหลักของ OOP อย่างครบถ้วน:
- **Encapsulation:** มีการแยกส่วนข้อมูลเป็น `private` และ `protected` และเรียกใช้งานผ่านฟังก์ชัน Getter/Setter เพื่อความปลอดภัยของข้อมูล
- **Inheritance (การสืบทอดคุณสมบัติ):** - Class `Diary` สืบทอดคุณสมบัติมาจาก Class `DiaryEntry`
  - Class `LockedDiary` สืบทอดคุณสมบัติหลักมาจาก Class `Diary` อีกทอดหนึ่งเพื่อประยุกต์ระบบความปลอดภัยล็อกรหัสผ่าน
- **Polymorphism & Method Overriding:** มีการใช้งานฟังก์ชันเสมือน (`virtual void`) และใช้โครงสร้าง `override` ในคลาสลูกเพื่อปรับเปลี่ยนรูปแบบการทำงานให้เข้ากับเงื่อนไขของระบบล็อกรหัสผ่าน
